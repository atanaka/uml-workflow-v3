# Contributing / コントリビューション

Thank you for your interest in contributing! / ご関心をお寄せいただきありがとうございます！

## How to Contribute / 貢献方法

### Reporting Issues / 問題の報告

- Use GitHub Issues to report bugs or suggest features
- Include: skill name, execution mode, error message, steps to reproduce

### Pull Requests

1. Fork this repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit changes: `git commit -m 'Add your feature'`
4. Push: `git push origin feature/your-feature`
5. Open a Pull Request

### Skill Development Guidelines / スキル開発ガイドライン

- Each skill must have a `SKILL.md` as the entry point
- Use bilingual headers (English / 日本語) where possible
- Follow the PIPELINE.md pattern for workflow integration
- Keep SKILL.md under 2,000 lines to prevent context issues
- Extract large templates to separate files with lazy loading

## Code of Conduct / 行動規範

Be respectful and constructive. / 敬意を持ち、建設的に。
