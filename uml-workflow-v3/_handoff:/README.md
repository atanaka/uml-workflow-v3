# `_handoff/` ディレクトリの使い方

このディレクトリは、Claude とのセッション間でコンテキストを引き継ぐための
**作業ログ・設計議論の保管場所** です。

## 目的

Claude のメモリは夜間バッチで更新されるため、セッション直後に新セッションを
始めても、直前の議論内容が自動的には引き継がれない場合があります。
このディレクトリにファイル形式で記録を残すことで、次セッション開始時に
確実にコンテキストを再構築できます。

## ファイル命名規則

```
YYYY-MM-DD_<トピック>_<バージョン or 種別>.<拡張子>
```

**例:**
- `2026-05-13_xmi-plan-v5.md` — 計画書本体
- `2026-05-13_papyrus-sample.png` — 参考画像・実機サンプル
- `2026-05-13_session-notes.md` — セッションメモ・議論ログ

## 標準ファイル

| ファイル | 役割 |
|---|---|
| `README.md` | 本ファイル。`_handoff/` の使い方説明 |
| `NEXT-SESSION.md` | **次セッション冒頭で Claude に渡す導入文**。常に最新版を保つ |
| `YYYY-MM-DD_*.md` | 各セッションの成果物（計画書・設計議論ログ・決定事項） |
| `YYYY-MM-DD_*.png` 等 | 参考資料（スクリーンショット・図表など） |

## 運用フロー

### セッション終了時（Akira）

1. このセッションの成果物をダウンロードして `_handoff/` に保存
2. `NEXT-SESSION.md` を更新（または新規作成）して次回の起点を明文化
3. 必要に応じて Git にコミット

### 次セッション開始時（Akira → Claude）

1. `NEXT-SESSION.md` の内容を Claude にコピペ、または該当ファイル群を添付
2. Claude が文脈を把握できたら、`NEXT-SESSION.md` 末尾の「次にやること」から開始

## Git 管理ポリシー

このディレクトリを Git に含めるかは状況に応じて判断:

- **公開リポジトリで設計プロセスも公開したい場合**: そのままコミット
  （note.com 記事のネタにもなる、設計の透明性が ModelCraft AI の差別化要素）
- **個人作業ログとしてプライベートに保ちたい場合**: `.gitignore` に `_handoff/` を追加

## 注意事項

- 機密情報（顧客固有情報・APIキー等）は **絶対に置かない**
- メモリ追記（`memory_user_edits`）との併用が望ましい
  - メモリ: 「v5 計画書がある」レベルの最小要約
  - `_handoff/`: 詳細な内容
- 古い `_handoff/` ファイルは `_archive/` サブディレクトリに移動するなど、
  定期的な整理を推奨
