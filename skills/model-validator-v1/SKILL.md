# Model Validator v1

Comprehensive validation tool for UML v1 workflow artifacts.

## Overview / 概要

This skill performs rigorous validation of all models generated by the UML v1 workflow, ensuring:
- Cross-model consistency
- Completeness of specifications
- Business rule compliance
- Traceability maintenance
- Quality standards adherence
- Readiness for code generation

**Key capabilities:**
- ✅ Validates all v1 model artifacts
- ✅ Checks cross-model consistency
- ✅ Verifies completeness
- ✅ Identifies conflicts and gaps
- ✅ Generates detailed reports
- ✅ Provides actionable recommendations
- ✅ **Multi-language validation reports (Japanese/English/Bilingual)** ⭐ NEW!
- ✅ **Inherits language from domain model** ⭐ NEW!

---

## Language Support / 言語サポート ⭐

### Overview / 概要

This skill generates validation reports in multiple languages, inheriting settings from the domain model to ensure consistency with project documentation.

**Supported Languages:**
- **Japanese (日本語)**: Validation reports in Japanese
- **English**: Validation reports in English
- **Bilingual (バイリンガル)**: Dual-language reports for international teams

### Language Scope

**What is localized:**
- ✅ Report headers and sections
- ✅ Validation messages (errors, warnings, info)
- ✅ Recommendations and suggestions
- ✅ Summary tables and statistics
- ✅ Quality assessment text

**What is NOT localized:**
- ✅ Entity names (always show original)
- ✅ Attribute names (always show original)
- ✅ File names and paths
- ✅ JSON/XML/PlantUML code snippets

### Language Inheritance

**Priority Order:**
1. **From domain-model.json** (highest priority)
   - Reads `metadata.language` if present
   - Ensures report matches model language
2. **Manual override** (if specified)
   - Can override via `report_language` parameter
3. **Auto-detection** (fallback)
   - Analyzes entity descriptions
   - Defaults to English if unclear

### Language Configuration

**Parameters:**
```python
language_options = {
    "report_language": "auto",          # auto | ja | en | bilingual
    "inherit_from_domain_model": True,  # Inherit language settings
    "error_messages": "auto",           # auto | ja | en
    "recommendations": "auto",          # auto | ja | en
    "technical_terms": "preserve"       # preserve | translate
}
```

## Validation Categories / 検証カテゴリ

### 1. Consistency Validation

**1.1 Actor Consistency**
- ✓ All actors in use cases exist in activity diagram
- ✓ All actors in activity diagram exist in domain model
- ✓ Actor names are consistent across models

**1.2 Entity Consistency**
- ✓ All entities referenced in use cases exist in domain model
- ✓ All entities in domain model are used in at least one use case
- ⚠️ Orphaned entities identified

**1.3 Relationship Consistency**
- ✓ All relationships in use cases match domain model
- ✓ Cardinality is consistent
- ✓ Navigability is defined

**1.4 Method Consistency (with sequence diagrams)**
- ✓ All messages in sequence diagrams correspond to methods in class diagram
- ✓ Method signatures match
- ✓ Return types align

**1.5 State Consistency (with state machines)**
- ✓ Status enums in domain model match state machine states
- ✓ Transitions in methods align with state machine
- ✓ Initial/final states are defined

---

### 2. Completeness Validation

**2.1 Domain Model Completeness**
- ✓ All entities have primary keys
- ✓ All entities have timestamps (createdAt, updatedAt)
- ✓ All relationships have cardinality
- ✓ All status fields have enumerations
- ✓ All business methods have signatures
- ⚠️ Missing attributes flagged

**2.2 Use Case Completeness**
- ✓ All use cases have actors
- ✓ All use cases have main success scenario
- ✓ All use cases have preconditions
- ✓ All use cases have success guarantee
- ⚠️ Missing extensions identified

**2.3 Business Rule Completeness**
- ✓ All business rules are referenced
- ✓ All constraints have implementations
- ✓ All validations are defined
- ⚠️ Unimplemented rules flagged

**2.4 Sequence Diagram Completeness**
- ✓ All use cases have sequence diagrams
- ✓ All main flows are diagrammed
- ✓ Critical extensions are diagrammed
- ⚠️ Missing diagrams identified

**2.5 State Machine Completeness**
- ✓ All stateful entities have state machines
- ✓ All state transitions are defined
- ✓ All guards are specified
- ⚠️ Invalid transitions identified

---

### 3. Business Rule Compliance

**3.1 Constraint Validation**
- ✓ All constraints in business rules are enforced in model
- ✓ Domain model attributes satisfy constraints
- ✓ State transitions respect constraints

**3.2 Validation Rule Coverage**
- ✓ All validation rules have corresponding model elements
- ✓ Validation logic is traceable to requirements

**3.3 Invariant Checking**
- ✓ Class invariants are maintained
- ✓ Relationship invariants are enforced

---

### 4. Traceability Validation

**4.1 Requirements Traceability**
- ✓ All business scenarios trace to activity diagrams
- ✓ All activity actions trace to use cases
- ✓ All use cases trace to domain entities
- ✓ All domain entities trace to code

**4.2 Coverage Analysis**
- ✓ All requirements are covered by use cases
- ✓ All use cases are implemented in domain model
- ✓ All domain model elements are used

**4.3 Impact Analysis**
- Changes to requirements → Affected use cases
- Changes to domain model → Affected use cases
- Changes to business rules → Affected validations

### 5. Quality Validation

**5.1 Naming Conventions**
- ✓ Entity names: PascalCase
- ✓ Attribute names: camelCase
- ✓ Method names: camelCase
- ✓ Consistent terminology (glossary compliance)

**5.2 Model Depth**
- ✓ Class diagram has appropriate depth (3-7 entities minimum)
- ✓ Use cases are appropriately sized (3-8 points)
- ✓ Sequence diagrams show sufficient detail

**5.3 Documentation Quality**
- ✓ All models have descriptions
- ✓ All business rules are documented
- ✓ All constraints have rationale

### Step 2: Run Consistency Checks

**2a. Actor consistency:**
```python
activity_actors = set(models['activity']['actors'])
usecase_actors = set(models['usecases']['actors'])
domain_actors = set(models['domain']['actors'])

# Check consistency
missing_in_usecases = activity_actors - usecase_actors
missing_in_domain = usecase_actors - domain_actors

if missing_in_usecases:
    report_error(f"Actors in activity diagram not found in use cases: {missing_in_usecases}")

if missing_in_domain:
    report_warning(f"Actors in use cases not found in domain model: {missing_in_domain}")
```

**2b. Entity consistency:**
```python
usecase_entities = extract_entities_from_usecases(models['usecases'])
domain_entities = set(e['name'] for e in models['domain']['entities'])

# Check all referenced entities exist
missing_entities = usecase_entities - domain_entities
if missing_entities:
    report_error(f"Entities referenced in use cases but not in domain model: {missing_entities}")

# Check for unused entities
unused_entities = domain_entities - usecase_entities
if unused_entities:
    report_warning(f"Entities in domain model not used in use cases: {unused_entities}")
```

**2c. Method consistency (with sequence diagrams):**
```python
if 'sequences' in models:
    for seq_diagram in models['sequences']['sequence_diagrams']:
        for message in seq_diagram['messages']:
            entity_name = message['to']
            method_name = extract_method_name(message['message'])
            
            # Find entity in domain model
            entity = find_entity(models['domain'], entity_name)
            if not entity:
                report_error(f"Entity '{entity_name}' in sequence diagram not in domain model")
                continue
            
            # Check if method exists
            methods = [m['name'] for m in entity.get('business_methods', [])]
            if method_name not in methods:
                report_error(f"Method '{method_name}' called in sequence diagram not defined in {entity_name}")
```

### Step 3: Run Completeness Checks

**3a. Domain model completeness:**
```python
for entity in models['domain']['entities']:
    # Check primary key
    has_id = any(attr.get('is_primary_key') for attr in entity['attributes'])
    if not has_id:
        report_error(f"Entity '{entity['name']}' missing primary key")
    
    # Check timestamps
    attr_names = [attr['name'] for attr in entity['attributes']]
    if 'createdAt' not in attr_names:
        report_warning(f"Entity '{entity['name']}' missing createdAt timestamp")
    if 'updatedAt' not in attr_names:
        report_warning(f"Entity '{entity['name']}' missing updatedAt timestamp")
    
    # Check status fields have enums
    for attr in entity['attributes']:
        if 'status' in attr['name'].lower() or 'state' in attr['name'].lower():
            if not is_enumeration(models['domain'], attr['type']):
                report_warning(f"{entity['name']}.{attr['name']} should use enumeration type")
```

**3b. Use case completeness:**
```python
for usecase in models['usecases']['usecases']:
    # Check required fields
    if not usecase.get('primary_actor'):
        report_error(f"Use case '{usecase['name']}' missing primary actor")
    
    if not usecase.get('main_success_scenario'):
        report_error(f"Use case '{usecase['name']}' missing main success scenario")
    
    if not usecase.get('preconditions'):
        report_warning(f"Use case '{usecase['name']}' missing preconditions")
    
    if not usecase.get('success_guarantee'):
        report_warning(f"Use case '{usecase['name']}' missing success guarantee")
    
    # Check extensions
    if not usecase.get('extensions'):
        report_info(f"Use case '{usecase['name']}' has no extension flows (may be intentional)")
```

**3c. Business rule implementation:**
```python
for rule in models['domain'].get('business_rules', []):
    # Check if rule is implemented
    implemented = False
    
    # Check in validation rules
    for entity in models['domain']['entities']:
        for attr in entity.get('attributes', []):
            if attr.get('validation', {}).get('rule_id') == rule['id']:
                implemented = True
                break
    
    # Check in business methods
    for entity in models['domain']['entities']:
        for method in entity.get('business_methods', []):
            if rule['id'] in method.get('applies_business_rules', []):
                implemented = True
                break
    
    if not implemented:
        report_warning(f"Business rule '{rule['id']}' ({rule['description']}) not implemented in model")
```

### Step 4: Run Business Rule Compliance Checks

**4a. Constraint validation:**
```python
for entity in models['domain']['entities']:
    for constraint in entity.get('constraints', []):
        # Parse constraint
        constraint_expr = constraint['expression']
        
        # Validate constraint is enforceable
        if not is_enforceable(constraint_expr, entity):
            report_error(f"Constraint '{constraint_expr}' on {entity['name']} cannot be enforced")
```

**4b. State transition validation (with state machines):**
```python
if 'statemachines' in models:
    for sm in models['statemachines']['state_machines']:
        entity_name = sm['entity']
        entity = find_entity(models['domain'], entity_name)
        
        # Check all transitions are valid
        for transition in sm['transitions']:
            from_state = transition['from_state']
            to_state = transition['to_state']
            trigger = transition['trigger']
            
            # Find method in entity
            method = find_method(entity, trigger)
            if not method:
                report_error(f"Transition trigger '{trigger}' not found as method in {entity_name}")
            
            # Check guards are enforceable
            for guard in transition.get('guards', []):
                if not is_enforceable(guard['condition'], entity):
                    report_warning(f"Guard '{guard['condition']}' may not be enforceable")
```

### Step 5: Run Traceability Checks

**5a. Requirements coverage:**
```python
# From business scenario to activity diagram
scenario_steps = extract_steps_from_scenario(models.get('scenario'))
activity_actions = [act['action'] for act in models['activity']['activities']]

uncovered_steps = []
for step in scenario_steps:
    if not any(step_matches_action(step, action) for action in activity_actions):
        uncovered_steps.append(step)

if uncovered_steps:
    report_warning(f"Business scenario steps not covered in activity diagram: {uncovered_steps}")

# From activity diagram to use cases
activity_actions_set = set(activity_actions)
usecase_actions = extract_actions_from_usecases(models['usecases'])

uncovered_actions = activity_actions_set - usecase_actions
if uncovered_actions:
    report_warning(f"Activity diagram actions not covered in use cases: {uncovered_actions}")

# From use cases to domain model
for usecase in models['usecases']['usecases']:
    entities_referenced = extract_entities_from_usecase(usecase)
    for entity_name in entities_referenced:
        entity = find_entity(models['domain'], entity_name)
        if not entity:
            report_error(f"Use case '{usecase['name']}' references non-existent entity '{entity_name}'")
```

### Step 6: Run Quality Checks

**6a. Naming convention validation:**
```python
for entity in models['domain']['entities']:
    # Entity name should be PascalCase
    if not is_pascal_case(entity['name']):
        report_warning(f"Entity name '{entity['name']}' should be PascalCase")
    
    # Attribute names should be camelCase

```

entity_count = len(models['domain']['entities'])

## Best Practices / ベストプラクティス

### For Quality

1. **Run validation before code generation**: Always validate models first
2. **Fix all errors**: Do not generate code with errors present
3. **Address critical warnings**: Review and fix high-priority warnings
4. **Maintain traceability**: Ensure complete coverage
5. **Document exceptions**: If warnings cannot be fixed, document why

### For Efficiency

1. **Automate validation**: Run as part of CI/CD pipeline
2. **Validate incrementally**: After each modeling step
3. **Review reports regularly**: Don't let issues accumulate
4. **Track quality metrics**: Monitor validation scores over time

### For Teams

1. **Establish quality gates**: Define acceptable error/warning thresholds
2. **Review together**: Discuss validation reports in team meetings
3. **Share best practices**: Learn from common validation failures
4. **Version control reports**: Track improvement over time

## Common Validation Failures / よくある検証エラー

### Scenario 1: Missing Primary Keys
**Error**: Entity missing primary key  
**Cause**: Entity created without ID attribute  
**Fix**: Add primary key attribute with `is_primary_key: true`

### Scenario 2: Method Not Found
**Error**: Sequence diagram calls non-existent method  
**Cause**: Method name mismatch or missing definition  
**Fix**: Add method to entity or correct method name in sequence diagram

### Scenario 3: Invalid State Transition
**Error**: State machine transition invalid  
**Cause**: Transition not allowed by business rules  
**Fix**: Update state machine or add guard condition

### Scenario 4: Orphaned Entity
**Warning**: Entity not used in any use case  
**Cause**: Entity defined but never referenced  
**Fix**: Remove entity or add use case that uses it

### Scenario 5: Incomplete Use Case
**Warning**: Use case missing preconditions  
**Cause**: Use case specification incomplete  
**Fix**: Add preconditions section to use case
